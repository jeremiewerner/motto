<objective>
Migrate the CI publish job from `NPM_TOKEN`-based auth to npm trusted publishing (OIDC): add `id-token: write`, drop the token env entirely, add `--provenance`. Extend the structural test suite to guard the new contract, and document the ship-time-only token-retirement follow-through (D-15/D-16/D-17) in the release skill so it isn't lost between now and the v0.0.6 ship tag.

Purpose: PUB-05 is the phase's core publish-hardening requirement. The static configuration (this plan) is fully verifiable now; the live OIDC handshake itself is explicitly deferred to the milestone's own ship tag (D-13) — this plan's job is to leave nothing ambiguous for that moment. Note: the npm-drift CI check will keep reporting yellow (0.0.3 registry vs. a higher package.json version) until that ship tag, because the 0.0.5 catch-up publish was formally abandoned (D-14) — that is expected, not a regression introduced by this plan.
Output: OIDC-authenticated publish job, a new structural test describe block, and a new "Step 9" in `skills/release/SKILL.md`.
</objective>

<objective>
Implement `validateSkill`, the pure skill-schema validator. It enforces the frontmatter fields,
name rules, body spine, and shared-reference safety, returning a uniform error array — never throws (D-01).

Signature (stable for Phase 2 — `lint` aggregates these across skills):
`validateSkill(skill, sharedRefs?: Set<string>) -> Array<{ skill: string, message: string }>`
where `skill = { dirName: string, data: object, body: string }` and `sharedRefs` is the set of
available shared-reference basenames without `.md` (defaults to an empty Set). Returns `[]` when valid;
each error's `skill` field is the `dirName`.

This operates on PLAIN OBJECTS only — no filesystem, no YAML parsing, no CLI. It does not import `yaml`.

Purpose: the schema is the product (CLAUDE.md core value). These checks are the strict rules that
guarantee a skill conforms before it ships.
Output: `src/schema.js`, `test/schema.test.js`.

- `src/schema.js`, `test/schema.test.js`
</objective>

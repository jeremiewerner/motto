<objective>
Implement the pure `motto.yaml` config validator. It parses raw config text and validates the identity
fields and plugin names, returning the config plus a uniform error array — never throws (D-01).

Signature (stable for Phase 2 — `loadConfig` there reads the file and passes its contents):
`loadConfig(text: string) -> { config: object, errors: Array<{ message: string }> }`
where `text` is the raw `motto.yaml` content. The file READ is Phase-2 plumbing; this function owns the
parse + validation logic it wraps. Returns the parsed `config` object (possibly partial on parse error)
and `errors[]` (empty when valid).

Purpose: enforces project-level config conformance (identity + plugin names) with the same strictness
and the same never-throw error model as the skill validator.
Output: `src/config.js`, `test/config.test.js`.

## Artifacts this plan produces
- `loadConfig` (exported from `src/config.js`)
- `src/config.js`, `test/config.test.js`
</objective>

<objective>
Establish the project scaffold and the first pure-core function, `parseFrontmatter`.

`parseFrontmatter(text)` splits a SKILL.md into frontmatter `data` + Markdown `body`, normalizing input first, and reports structural malformation through a uniform `errors[]` array — it NEVER throws (D-01, D-03).

Signature (stable for Phase 2 — `discoverSkills` will call it per file):
`parseFrontmatter(text: string) -> { data: object, body: string, errors: Array<{ message: string }> }`

Phase 2 attaches the skill name to each `{ message }` when composing lint output; this function has only the raw text and therefore returns `{ message }` errors (no `skill` field).

Purpose: This is the architectural spine of the pure core — the uniform never-throw `errors[]` model so nothing can crash the linter downstream.
Output: amended `package.json`, `src/frontmatter.js`, `test/frontmatter.test.js`.

<objective>
Extend `loadConfig()` in `src/config.js` to validate an OPTIONAL `mottoVersion` field: absence is never an error (bootstrap/pre-stamp compatibility, VER-04 support), but a present-and-malformed value is a clean `errors[]` entry (never a throw). Prove the never-throw boundary with an adversarial malformed-input matrix in `test/config.test.js` (VER-05).

Purpose: Malformed stamps are a data-shape problem of the same kind as a missing required field, so they belong in config.js's existing validation layer as `errors[]` (D-R1), making `ok:false` — distinct from skew (a warning) and from absence (no-op). This upholds the project's never-throw invariant (recurred 5×) with mandatory adversarial coverage.
Output: modified `src/config.js`, extended `test/config.test.js`, fully green.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| source tree -> published tarball | The `files` allowlist decides which local paths cross into the public npm tarball. Anything mis-included leaks publicly. |
| developer machine -> npm registry | `npm publish` uploads the tarball under the maintainer's credentials to registry.npmjs.org. |
| local git -> remote origin | `git push --follow-tags` pushes the release commit and tag to `github:jeremiewerner/motto`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-01 | Information Disclosure | npm tarball composition (`files` allowlist) | high | mitigate | D-05 scripted `npm pack --dry-run --json` assertion in release Step 4 hard-fails on any path outside `bin/`/`src/`/`dist/public/` (+ auto package.json/README/LICENSE), preventing `skills/`, `test/`, `.planning/` leakage (NPM-04). Backstop: `files` allowlist is additive (Task 1). |
| T-07-02 | Spoofing | npm registry auth at publish time | high | mitigate | Release Step 5 requires `npm whoami` verification (STOP on error or wrong account) before `npm publish`, preventing publish under expired/wrong credentials or an unintended account. |
| T-07-03 | Tampering | npm lifecycle scripts (`prepublishOnly`, `version`) | medium | mitigate | Scripts are constrained to `node bin/motto.js build` and a local motto.yaml edit + `git add`; no network calls, no external/downloaded scripts. Scripts reviewed in Task 2; single runtime dep unchanged. |
| T-07-04 | Tampering | stale/unbuilt `dist/public/` in tarball | medium | mitigate | `prepublishOnly` rebuilds `dist/public/` on `npm publish` (D-03); release Step 3 dogfood build precedes Step 4 verify (prepublishOnly does not fire on `--dry-run`), so the verified artifact is current. |
| T-07-05 | Tampering | publish target registry | low | accept | No `.npmrc` registry override in the project; npm defaults to registry.npmjs.org. No action needed. |
| T-07-06 | Spoofing | package-name typosquatting | low | accept | Scoped name `@jeremiewerner/motto` is namespace-protected; the unscoped `motto` name is irrelevant to this package. |
| T-07-SC | Tampering | npm/pip/cargo installs | low | accept | No new packages are installed in this phase (RESEARCH Package Legitimacy Audit: zero new deps). No legitimacy checkpoint required. |
</threat_model>

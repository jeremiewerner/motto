<objective>
Rewrite the maintainer `release` skill so the local flow stops at `git push --follow-tags`, handing the publish responsibility (and the D-05 tarball assertion) to CI, and add the CI-handoff / verification / recovery sections that close the "tag pushed = assumed published" drift gap that actually put npm at 0.0.3 while tags ran ahead.

Purpose: The old skill made the maintainer the publisher (npm whoami / npm publish locally) and carried the D-05 check as an inline heredoc. Now that CI owns both (plan 21-01), the skill must shrink to the local half (tests → bump → push) and grow an explicit close-the-loop verification plus a never-re-tag recovery runbook — otherwise removing the local publish step makes the silent-drift failure mode more likely, not less (RESEARCH.md Pitfall 5).
Output: A rewritten `skills/release/SKILL.md` that still lints clean as a `template: procedure` skill, with the local publish/whoami steps removed and CI-handoff + Verify-CI-Published + If-CI-Publish-Fails sections added.
</objective>

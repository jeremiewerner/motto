<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| maintainer → GitHub Actions secret store | A long-lived npm credential is minted and injected into CI's encrypted secret store |
| GitHub Actions secret store → publish job | The secret is decrypted into the publish job env at run time (`NODE_AUTH_TOKEN`) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-21-07 | Tampering / Information Disclosure | NPM_TOKEN is a long-lived static credential — a standing attack surface for the milestone | high | mitigate | Use a GRANULAR, package-scoped (`@jeremiewerner/motto`), publish-only token with an expiry (not a classic account-wide token); store only as an encrypted GitHub Actions secret, never in a file/log; time-boxed — Phase 22 / PUB-05 migrates to OIDC trusted publishing (zero long-lived tokens) immediately after the public flip. |
| T-21-08 | Information Disclosure | Token value leaking during creation/storage (pasted into a file, commit, or chat) | high | mitigate | Only the secret name is ever recorded; the value is entered directly into `gh secret set` / GitHub UI; explicit instruction not to paste it anywhere else. |
| T-21-SC | Tampering | npm/pip/cargo installs | low | accept | Zero new packages; this plan installs nothing and edits no code. |
</threat_model>

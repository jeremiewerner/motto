<objective>
Close the single blocking gap from 21-VERIFICATION.md (Truth #6 / 21-REVIEW.md CR-01): the publish job's two idempotency guards key on different identifiers (npm_guard on package.json version, gh_guard/`gh release create` on the tag name) and are never cross-checked, so a tag pushed on a commit whose package.json version does not match (a plausible operator slip) produces a green CI run that publishes nothing to npm yet still creates a phantom GitHub Release — the exact npm/tag drift this phase was built to close.

Add one hard-fail guard step to `.github/workflows/ci.yml` that asserts the pushed tag matches package.json's version before either idempotency guard runs, plus a committed ordering-aware structural test that fails if the guard is ever removed or reordered, plus a minimal accuracy update to the release skill's runbook so its failure-mode guidance is not silently wrong for this new hard-fail class.

Purpose: The phase goal promises publishing "automatically and **safely**." This specific, documented operator-error path is not currently safe. This is the minimal closure — one guard step (already drafted verbatim in CR-01) plus an adversarial regression test.
Output: A `version_guard` step in the publish job (before `npm_guard`), a new `test/ci-workflow.test.js` structural test asserting presence AND ordering, and two surgical lines in `skills/release/SKILL.md` (Step 5 + Step 7).
</objective>

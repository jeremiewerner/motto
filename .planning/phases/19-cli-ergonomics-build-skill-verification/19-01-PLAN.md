<objective>
Add `--quiet` and `--format text|json` flags to `motto lint`/`motto build`, and move every `✗` error line (lint, build, and init) from stdout to stderr, so the CLI emits machine-readable, pipe-safe output that Phase 20's CI jobs can assert against. All work is confined to the presentation layer (`bin/motto.js`) plus its test file; the never-throw core (`src/lint.js`, `src/build.js`, `src/schema.js`) is untouched.

Purpose: Phase 20's pack-install E2E and dogfood CI jobs consume `--quiet` (silent stdout + exit 0) and `--format json` (parse stdout, check `.ok`) as contracts. This plan produces those contracts and closes Pitfall 7 (stdout/stderr conflation) so `motto lint --format json | jq .` never breaks on stray text.

Output: Modified `bin/motto.js` and extended `test/cli.test.js`.
</objective>

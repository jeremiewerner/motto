<objective>
Close the schema-strictness gap in `src/schema.js`: make `validateSkill` fully conformant
with the CLAUDE.md SKILL.md-frontmatter spec for `name` and `description` by adding exactly
three new checks (per locked decisions D-03/D-04/D-05):

1. `description` max **1024** chars — error only when description is present.
2. `description` no XML tags — flag pattern `/<[^>]+>/`, only when description is present (D-05).
3. `name` max **64** chars — slotted into the existing name cascade.

`name` no-XML needs no new check (the existing `NAME_KEBAB` regex already forbids `<`/`>` — D-04).

Purpose: The strict schema + linter is Motto's core value. Two of the three CLAUDE.md
length/XML rules are currently unenforced; this makes the validator spec-complete for the two
authoritative frontmatter fields.

(SELF-01..05, DOG-01..04) are already complete; no requirement ID is assigned to schema
strictness. Delivery is verified by the new unit tests, not a requirement-ID map.

Output: An extended `validateSkill` (pure function, never-throws, errors[] model preserved per
D-01) and six new test cases (B14–B19) in `test/schema.test.js`. No CLI, build, or dependency
changes (out of scope per D-01/D-02).

    `if (!data.description) { err("description is required"); }` block to add an `else`

        locked in D-05; do not alter it. A literal `<placeholder>` in a description trips it —
        acceptable per D-05.

<objective>
Close DEBT-04: README.md line 173 references the release skill as `/motto:release`, which is provably wrong regardless of installation. `skills/release/SKILL.md` is `audience: private`, so `motto build` routes it into `dist/private/` under the plugin name `motto-private` (per `motto.yaml` `plugins.private: motto-private`). Claude Code's skill invocation namespace is `<plugin-name>:<skill-name>`, so the release skill is never `/motto:release`. The correct form is either `/motto-private:release` (only if the maintainer has a working local-load path for the private plugin) or a plain file-path reference to `skills/release/SKILL.md` (if not).

Because whether `/motto-private:release` resolves depends on the maintainer's own machine (this repo has no `.claude/skills/` symlink and no `motto-private` entry in `.claude-plugin/marketplace.json`), a `checkpoint:human-verify` gate confirms the fact before committing to either fix.

Purpose: The README must not advertise a slash command that does not resolve.
Output: A corrected line 173 reference, chosen based on the maintainer's verified setup.
</objective>

## Cross-Phase Seams

### Seam 1: Phase 4 content → Phase 6 strictness (validateSkill)

**Risk:** Phase 6 added three new validation checks (name ≤64, description ≤1024, description no-XML). If the real skill tree's content violated any new rule, `motto lint` would exit non-zero, breaking DOG-01 and the dogfood test.

**Verified live:** `motto lint` exits 0 after Phase 6 lands. Phase 06 SUMMARY confirms actual lengths: names 7/13/17 chars (limit 64), descriptions 146/158/166 chars (limit 1024), no XML in any description. No false-trigger.

**Status: WIRED**

### Seam 2: Phase 4 skill names → Phase 5 dogfood test (after quick rename)

**Risk:** The quick rename task changed all three skill folder names (`authoring-a-skill→author-skill`, `motto-project-setup→setup-project`, `motto-release→release`) AFTER Phase 5 wrote `test/dogfood.test.js` with hardcoded `stat()` paths. Stale paths would cause the dogfood test to pass the build but fail artifact assertions.

**Verified by reading `test/dogfood.test.js`:** All `stat()` calls reference the new names (`author-skill`, `setup-project`, `release`). Quick task SUMMARY explicitly commits `test/dogfood.test.js` as Task 2 (commit 3d7972d). `node --test` green (71/71) confirms no stale path.

**Status: WIRED**

### Seam 3: DOG-04 NAME_KEBAB parity (schema.js ↔ config.js)

**Risk:** The two regex literals could silently diverge on future edits if the parity test was not wired correctly.

**Verified by grep:** Both files export `NAME_KEBAB = /^[a-z][a-z0-9]*(-[a-z0-9]+)*$/` (identical `.source` and `.flags`). `dogfood.test.js` imports both and asserts `.source` + `.flags` via `strictEqual`. The parity test is the first describe block in the dogfood file (no I/O dependency), so it fails fast.

**Status: WIRED**

### Seam 4: Husky pre-commit → npm test → dogfood test

**Risk:** DOG-03 claims "every commit" protection. If `.husky/pre-commit` didn't invoke `npm test`, or `npm test` didn't discover `test/dogfood.test.js`, the guard would be a documentation claim only.

**Verified:** `.husky/pre-commit` contains `npm test`. `package.json scripts.test` runs `node --test` which auto-discovers `*.test.js` recursively (Node 20 behavior). `test/dogfood.test.js` is in the `test/` directory. Quick task SUMMARY explicitly notes the pre-commit hook ran and passed on both rename commits.

**Status: WIRED**

### Seam 5: dist/ verbatim copy

**Risk:** DOG-02 requires SKILL.md verbatim in dist. A transform step could silently alter content.

**Spot-check:** First 5 lines of `dist/public/author-skill/SKILL.md` match `skills/author-skill/SKILL.md` byte-for-byte. Private `release/` has no `references/` directory (confirmed: `ls dist/private/release/` returns only `SKILL.md`).

**Status: WIRED**

# Requirements: Motto — v0.0.1 (ARCHIVED, SHIPPED 2026-06-30)

**Core Value:** A strict schema + linter that guarantees authored skills conform before they ship, then packages them into self-contained standard Agent Skill plugins.

All 22 requirements below shipped and were verified (per-phase VERIFICATION.md + milestone audit). Outcome: **all validated**, none adjusted or dropped.

## Requirements (all complete)

### Parsing (PARSE)
- [x] **PARSE-01**: `parseFrontmatter` extracts YAML `data` + Markdown `body`; missing `--- ... ---` reported as a named error (never-throw, per D-03)
- [x] **PARSE-02**: Input normalized (CRLF→LF, BOM stripped) before regex/YAML parse
- [x] **PARSE-03**: YAML parse errors surface as lint errors (malformed skill reported, never dropped)
- [x] **PARSE-04**: Stray `---` inside frontmatter detected and reported

### Linting / Schema (LINT)
- [x] **LINT-01**: Required frontmatter fields validated — `name`, `description`, `audience`
- [x] **LINT-02**: `name` letter-start kebab (`/^[a-z][a-z0-9]*(-[a-z0-9]+)*$/`), equals folder, no `anthropic`/`claude`
- [x] **LINT-03**: `audience` is exactly `public` or `private`
- [x] **LINT-04**: Body spine — first non-blank line is `# ` H1, body contains a `**Role:**` line
- [x] **LINT-05**: Each `shared_references` entry is a safe basename resolving to `shared/references/<name>.md`
- [x] **LINT-06**: All errors across all skills collected as `skill: message`; exits 0 (clean) / 1 (errors)
- [x] **LINT-07**: Discovery deterministic (sorted by name) and per-file error-isolated

### Config (CONF)
- [x] **CONF-01**: `motto.yaml` loaded + validated; `name`, `version`, `description`, `plugins.public` required (reported together)
- [x] **CONF-02**: `plugins.public` (and `plugins.private` when present) match the kebab plugin-name regex
- [x] **CONF-03**: `plugins.private` is optional

### Build / Packaging (BUILD)
- [x] **BUILD-01**: `motto build` runs lint first; on failure writes nothing, surfaces lint diagnostics
- [x] **BUILD-02**: `dist/` wiped, each skill copied verbatim into `dist/<audience>/<name>/` (symlinks preserved)
- [x] **BUILD-03**: Declared `shared_references` bundled into each skill's `references/`
- [x] **BUILD-04**: Each bucket gets `.claude-plugin/plugin.json` from `motto.yaml`, always including `version`
- [x] **BUILD-05**: Private bucket emitted only when private skills exist and `plugins.private` is set
- [x] **BUILD-06**: Skills are siblings of `.claude-plugin/` in dist (never nested)

### CLI (CLI)
- [x] **CLI-01**: `motto lint` validates the current project, prints per-error output with correct exit code
- [x] **CLI-02**: `motto build` builds the current project, reports output dir, reuses lint diagnostics on failure

## Out of Scope (unchanged)

| Feature | Reason |
|---------|--------|
| `--fix` auto-repair | Can't meaningfully auto-generate a Role/structure; risks corrupting intent |
| Watch mode | YAGNI for a build-on-demand tool |
| Per-directory config overrides | Would undermine single-schema enforcement |
| Content transform / inlining compiler | Fights the maintainability principle; bundling is copy-only |
| MCP-dependency resolution | `dependencies` declared/linted but not resolved in v1 |
| Bespoke skill format / custom runtime | Output must stay portable standard Agent Skills |

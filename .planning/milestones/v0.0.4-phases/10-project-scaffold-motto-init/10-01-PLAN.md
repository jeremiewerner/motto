Create `src/init.js` following the never-throw orchestrator house style of `src/build.js`. Import `mkdir`, `writeFile`, `readdir` from `node:fs/promises`; `basename`, `join` from `node:path`; `execFileSync` from `node:child_process`; and `NAME_KEBAB` from `./schema.js` (single source per D-09/SC4 — do NOT import from config.js and do NOT redefine the regex).

Export `async function scaffoldProject(targetDir, { name, force = false } = {})` that returns a structured object and never throws at its boundary. Order of operations is validate-then-guard-then-write (Pitfall 1):

1. Compute the effective name as the passed `name` when present, otherwise `basename(targetDir)` (INIT-01: `[name]` fills fields, defaults to cwd basename). Validate `NAME_KEBAB.test(effectiveName)` FIRST, before constructing any template string (INIT-05/D-09, Pitfall 1). On failure return `{ ok: false, reason: 'invalid-name', name: effectiveName, suggestion: suggestKebabName(effectiveName), errors: [ ... ] }` where the single error message names the exact schema rule violated (letter-start kebab-case, regex `/^[a-z][a-z0-9]*(-[a-z0-9]+)*$/`). Per D-08 the error surfaces the rule plus a sanitized suggestion; never auto-rename.

2. Unless `force` is true, call `listNonIgnorableEntries(targetDir)`; if it returns any entries, return `{ ok: false, reason: 'not-empty', offending, errors: [] }` (INIT-04/D-05). `--force` ONLY skips this check — it must never call `rm`/`rmdir` (D-06, Pitfall 3: init contains zero delete operations).

3. Resolve `owner = resolveGitOwnerName()` (INIT-03) — best-effort, never throws.

4. Call `writeScaffold(targetDir, { name: effectiveName, owner })` and return `{ ok: true, created, errors: [] }` where `created` is the list of relative paths written.

Internal helpers:

- `IGNORABLE_ENTRIES = new Set(['.git', '.DS_Store'])` and `async function listNonIgnorableEntries(targetDir)`: mirror `discoverSkillNames` — `readdir(targetDir, { withFileTypes: true })` inside try/catch returning `[]` on ENOENT (re-throw only non-ENOENT), map to names, filter out `IGNORABLE_ENTRIES`, sort with `localeCompare` for deterministic capped output (D-05, D-07).

- `function resolveGitOwnerName()`: `execFileSync('git', ['config', 'user.name'], { encoding: 'utf8' }).trim()` inside try/catch; return the trimmed name, or the placeholder `'Your Name'` when empty/unset/git-missing (INIT-03 fallback, Pattern 3). Use `execFileSync` with an argument array — never `execSync` with a shell string (no shell spawned).

- `function suggestKebabName(raw)`: lowercase, replace runs of non-`[a-z0-9]` with a single `-`, trim leading/trailing `-`, strip any leading non-letter, fallback to `'my-project'` if the result is empty (D-08 hint generator; `suggestKebabName('My Project')` must yield `'my-project'`).

- `async function writeScaffold(targetDir, { name, owner })`: `mkdir({ recursive: true })` each needed directory then `writeFile` (overwrites by default — this IS the D-06 force-overwrite behavior, no delete needed) for the fixed scaffold paths. The validated name is used only as file CONTENT, never as a path segment. Emit these inline-string templates:
  - `skills/hello-world/SKILL.md` — frontmatter `name: hello-world` (D-01), a non-empty `description:` with NO XML-tag shapes and 1024 chars or fewer, `audience: public` (D-03), NO `shared_references` key (D-04). Body: an H1 as the first non-blank line (e.g. `# Hello World`) and a line beginning with the bold Role marker, plus one or two plain sentences — MINIMAL, do NOT embed Motto documentation (D-02).
  - `shared/references/.gitkeep` — empty file (D-04; git does not track empty dirs — Pitfall 5).
  - `motto.yaml` — `name: <effectiveName>`, `version: "0.1.0"`, a generic one-line `description:`, and `plugins:` with `public: <effectiveName>` ONLY (no `plugins.private` — scaffold ships one public starter skill). Derive BOTH `name` and `plugins.public` from the SAME validated effective-name value (Pitfall 2).
  - `.gitignore` — lines `node_modules/` and `dist/private/`. Do NOT emit a bare `dist/` line and do NOT copy this repo's GSD-specific ignores; `dist/public/` must remain git-trackable so the marketplace.json relative pointer resolves for consumers (INIT-06).
  - `.claude-plugin/marketplace.json` — emit via `JSON.stringify(obj, null, 2) + '\n'` with top-level `name: <effectiveName>`, `owner: { name: <owner> }` (NO `email` key — omit entirely), and `plugins: [{ name: <effectiveName>, source: "./dist/public/", description: <same generic description as motto.yaml> }]`. Use the BARE relative-path string for `source` — NOT the `{ source: "npm", package: ... }` object form this repo uses, and NO `skills`/`strict` keys (INIT-03, Pattern 4).

Keep every helper never-throw at its own boundary. No `RESERVED`/64-char checks for the project name — `loadConfig` enforces only `NAME_KEBAB` on plugin names, so matching more would over-reject (Pattern 2, Anti-Patterns).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user terminal → CLI (`[name]` arg, cwd basename) | The only user-controlled input; flows into file CONTENTS (YAML/JSON values), never into path segments |
| CLI → git subprocess (`git config user.name`) | External process invoked for owner lookup; may fail (git missing / key unset) |
| CLI → filesystem (scaffold writes under targetDir) | Fixed set of relative paths; overwrite-by-default, never delete |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-10-01 | Tampering | Name → motto.yaml / marketplace.json content | high | mitigate | `NAME_KEBAB.test(effectiveName)` gates BEFORE any template interpolation (Pitfall 1); the `[a-z][a-z0-9-]*` character class makes YAML/JSON-breaking characters (`:`, `"`, newline) structurally unreachable |
| T-10-02 | Tampering | `git config` invocation in resolveGitOwnerName | medium | mitigate | `execFileSync('git', ['config', 'user.name'])` array-args form — no shell spawned, zero metacharacter surface; wrapped in try/catch that never throws |
| T-10-03 | Tampering / Elevation of Privilege | Scaffold write paths | high | mitigate | All scaffold paths are hardcoded relative to `targetDir`; the validated name is used only as file content, never a path segment. `NAME_KEBAB` additionally rejects `/` and `.`, so path traversal via `[name]` is impossible by construction |
| T-10-04 | Tampering (accidental data loss) | Non-empty-dir overwrite | high | mitigate | Refuse by default (D-05 allowlist), explicit `--force` required (D-06), and `--force` skips only the CHECK — init contains zero `rm`/`rmdir` operations (Pitfall 3) |
| T-10-SC | Tampering | npm/pip/cargo installs | high | accept | No packages installed this phase — stdlib only (RESEARCH Package Legitimacy Audit: empty). No supply-chain surface introduced |
</threat_model>

<objective>
Close the single Phase 10 verification gap (Truth #7 / REVIEW.md WR-01): restore
`scaffoldProject`'s documented never-throw contract for its two reproduced throw
paths, and add adversarial regression tests that lock the contract in place.

Purpose: `src/init.js`'s header + JSDoc promise a "Never-throw orchestrator"
returning `{ ok, errors }`, but two fs failure modes escape that contract and,
because `bin/motto.js` awaits the call with no try/catch, surface to the end user
as a raw Node stack trace. This is the exact class of never-throw hole flagged as
a recurring failure class in this project.

Output: `src/init.js` STEP 2 and STEP 4 wrapped in try/catch mapping fs errors to
`{ ok:false, errors:[...] }`; two new adversarial tests in `test/init.test.js`.
</objective>

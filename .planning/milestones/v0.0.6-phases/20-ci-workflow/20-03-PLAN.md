| Boundary | Description |
|----------|-------------|
| Fork PR -> CI runner | untrusted PR code executes in the workflow (post-public-flip concern; private today) |
| Workflow -> third-party action tags (`actions/*@v6`) | mutable major-version tags resolved at run time |
| GITHUB_TOKEN -> repo | token permissions scope what workflow code can touch |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-20-06 | Elevation of Privilege | PR trigger | high | mitigate | Use `pull_request` (read-only fork-scoped token, no secrets), never `pull_request_target` (D-01 + RESEARCH Security Domain); acceptance criterion asserts the string is absent |
| T-20-07 | Elevation of Privilege | workflow token permissions | medium | mitigate | Explicit top-level `permissions: contents: read` — least-privilege baseline before Phase 21 adds write scopes to one job only |
| T-20-08 | Tampering | `actions/checkout@v6` / `actions/setup-node@v6` tag pins | medium | accept | First-party GitHub actions pinned by major tag; SHA-pinning documented as the stricter alternative, deliberately not adopted at single-maintainer risk tolerance (recorded tradeoff, not an oversight) |
| T-20-SC | Tampering | npm/pip/cargo installs | n/a | accept | Zero new packages this phase; `npm ci` in jobs installs only committed-lockfile deps |

|----------|-------------|
| `outputs:` path → filesystem | An author-declared output path is resolved and stat'd; a symlink inside the skill dir can point outside it |
| other skills' SKILL.md → audience pre-pass | `loadSkillAudiences` re-reads every skill's frontmatter (untrusted, possibly malformed) to build the audience map |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-15-02 | Tampering / Information Disclosure | `checkOutputsFs` symlink resolution | high | mitigate | `realpath` both skill dir and target, then `path.relative` containment — a symlinked file escaping the skill dir fails lint; proven by an adversarial symlink fixture (Task 3) |
| T-15-01 | Tampering / Information Disclosure | `checkOutputsFs` fs-eligibility gate | high | mitigate | Reuses `isOutputPathLexicallySafe` so a lexically-unsafe entry never reaches `stat`/`realpath` — traversal/absolute paths are gated out before any fs access |
| T-15-04 | Denial of Service | `loadSkillAudiences`, `checkOutputsFs` | medium | mitigate | Both helpers never-throw: the pre-pass swallows per-file failures silently; the fs helper's stat/realpath are try/caught, backstopped by processSkill's D2-06 catch-all (VAL-06) |
| T-15-05 | Tampering (TOCTOU) | lint→build window on `outputs:` files | low | accept | `motto lint` is a static pre-flight tool, not a runtime boundary; a stat/realpath→build race needs local write access during the window — outside this single-maintainer local-CLI threat model (RESEARCH Security Domain) |
